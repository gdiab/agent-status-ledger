# ASL v0.2 Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Quiet the morning report's top (noise filter + silent retune) and enrich its detail (error context + status tooltips/legend), per `docs/superpowers/specs/2026-07-09-v02-polish-design.md`.

**Architecture:** Four independent polish changes on the existing pipeline. Connectors gain an `awaitingUser` flag and error-context composition at ingest; `inferStatus` demotes long-quiet ball-in-your-court sessions to `idle`; `buildReport` filters trivial profiles into a new optional `Report.trivialProfiles` field; renderers add a footer line, hover tooltips, and a shared legend. `schemaVersion` stays 1 (additive optional field only).

**Tech Stack:** Bun + TypeScript. `bun test`, `bunx tsc --noEmit`. No new dependencies.

## Global Constraints

- Work on branch `v02-polish` off `main` (create in Task 1, Step 0).
- `schemaVersion` stays `1`. `Report` may only gain the optional `trivialProfiles?: string[]` field. No other type removals/renames.
- `FactSheet.errors` stays `string[]` — error context is composed INTO the string at ingest.
- TDD: every behavior change starts with a failing test; run the test and see it fail before implementing.
- Commit messages: conventional style (`feat:`, `fix:`, `test:`, `docs:`); never mention Claude and no Co-Authored-By lines.
- Full gate per task: `bun test` all green, then `bunx tsc --noEmit` clean.
- Golden files (`fixtures/golden/report.md|json`) are regenerated ONLY in Task 8 via `UPDATE_GOLDEN=1`, with the diff inspected and explained in the commit message. If an intermediate task breaks `tests/golden.test.ts`, regenerate there instead, inspect the diff, and say so in that task's commit.
- Unknown/unparseable log shapes must fail toward alerting: absent `awaitingUser` ⇒ treated as `false` ⇒ `silent` preserved.

```ts
const CONTEXT_MAX = 80;

// "exit code 143 — while Bash: xcrun simctl…" — gives the reader (and the
// narrative LLM) what the agent was doing when the error happened. Input is
// flattened and truncated; redaction runs later on the composed string.
export function withContext(message: string, toolName: string, input: unknown): string {
  const raw = input === undefined || input === null ? "" : typeof input === "string" ? input : JSON.stringify(input);
  const flat = raw.replace(/\s+/g, " ").trim();
  if (!flat) return `${message} — while ${toolName}`;
  const slice = flat.length > CONTEXT_MAX ? `${flat.slice(0, CONTEXT_MAX)}…` : flat;
  return `${message} — while ${toolName}: ${slice}`;
}
```

```ts
// A 12-second accidental session run from ~ shouldn't get a card, let alone
// an urgent exception — but nothing disappears silently: names are reported
// in Report.trivialProfiles and rendered as a footer line.
export function isTrivialProfile(profile: AgentProfile, commits: CommitEvidence[], minSessionSeconds: number): boolean {
  return (
    profile.sessions.every(
      (s) =>
        Date.parse(s.lastEventAt) - Date.parse(s.startedAt) < minSessionSeconds * 1000 &&
        s.filesTouched.length === 0 &&
        s.errors.length === 0,
    ) && !commits.some((c) => c.attributed)
  );
}
```
